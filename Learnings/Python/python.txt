2)Multithreading vs Multiprocessing
    Multiple threads in one process. It uses Shared memory.
    Best for API calls, File download, Database calls

    Multiple processes. It uses the Separate memory. Uses multiple CPU cores.
    Best for Image processing, Video encoding, Machine learning.

    Multithreading uses multiple threads within the same process, 
    sharing memory, and is best for I/O-bound tasks such as API calls or 
    file downloads. Multiprocessing uses multiple independent processes with 
    separate memory, allowing true parallel execution across CPU cores, making 
    it ideal for CPU-bound tasks like image processing or data analysis.

7)Synchronous vs Asynchronous
    Synchronous execution waits for one task to complete before starting the 
    next. Asynchronous execution allows the program to switch to other tasks 
    while waiting for I/O operations, improving throughput and responsiveness. 
    
    Async is especially useful for APIs, database calls, and network requests.

    Async:
        import asyncio

        async def api_call():
            print("API Started")
            await asyncio.sleep(3)
            print("API Finished")

        async def other_task():
            for i in range(3):
                print(f"Working {i}")
                await asyncio.sleep(1)

        async def main():
            await asyncio.gather(
                api_call(),
                other_task()
            )

        asyncio.run(main())

    sync:
         import time

        def api_call():
            print("Calling API...")
            time.sleep(3)      # Simulate API delay
            return "Data received"

        def get_data():
            data = api_call()
            print(data)

        print("Start")
        get_data()
        print("End")

    Sync = Wait → Then continue
    Async = Start → Don't wait → Do other work → Come back when ready

8)decorator
    A decorator is a function that wraps another function to add extra behavior without modifying the original function's code.
    It is commonly used for logging, authentication, authorization, caching, and performance measurement.

    logger is a function that accepts another function as an argument.

    def logger(func):

        def wrapper():
            print("Logging Started")
            func()
            print("Logging Finished")

        return wrapper


    @logger
    def login():
        print("Login Successful")

9)*args vs **kwargs

    The easiest way to remember is:
    *args → Multiple positional arguments → Stored as a tuple
    **kwargs → Multiple keyword arguments → Stored as a dictionary

    def add(*args):
        total = 0

        for num in args:
            total += num

        print(total)

    add(10, 20)
    add(10, 20, 30)
    add(10, 20, 30, 40)

    def student(**kwargs):

        for key, value in kwargs.items():
            print(key, ":", value)

    student(
        name="Rahul",
        age=25,
        city="Bangalore"
    )

10)Exception Handling
    Exception handling is a mechanism in Python to handle runtime errors 
    gracefully without stopping the execution of the program. 
    It is done using try, except, else, and finally blocks.

    try:
    # Code that may cause error

    except:
        # Handle error

    else:
        # Runs if no exception occurs

    finally:
        # Always executes

    @app.route("/user")
    def get_user():

        try:
            user = database.get_user()
            return user

        except DatabaseError:
            return {
                "error": "Database unavailable"
            }, 500

        finally:
            database.close()

    raise vs assert
        raise is used to manually throw an exception when a specific error condition occurs.
        You explicitly tell Python: "An error has happened, stop and handle it."
        we intentionally raised an exception because the business rule failed.

        assert is a debugging tool used to check whether a condition is true. 
        If the condition is false, Python raises an AssertionError.

        def withdraw(amount, balance):

            if amount > balance:
                raise ValueError("Insufficient balance")

            print("Withdrawal successful")

        withdraw(1000, 500)

        assert syntax: assert condition, "error message"

        age = 15
        assert age >= 18, "Age should be 18 or above"
        print("User allowed")

    assert → "This should never happen. If it happens, my code has a bug."
    raise → "This can happen. Handle this error properly."

20)What is REST API?

    where clients communicate with servers using HTTP methods like GET, POST, PUT, DELETE to perform operations on resources.

    | Method | Purpose        | Example           |
    | ------ | -------------- | ----------------- |
    | GET    | Read data      | Get users         |
    | POST   | Create data    | Create user       |
    | PUT    | Full update    | Replace user      |
    | PATCH  | Partial update | Update email only |
    | DELETE | Remove data    | Delete user       |

21)Status Code
    200 - success
    201 - created 
    400 - bad request
    401 - not authenticated
    403 - not authorized
    404 - not found
    422 - validation error
    500 - internal server error
    504 - gateway timeout

    5xx Server Error Codes
        | Status Code | Name                  | Meaning                | Real Example          |
        | ----------- | --------------------- | ---------------------- | --------------------- |
        | **500**     | Internal Server Error | Application crashed    | Python exception      |
        | **501**     | Not Implemented       | Feature not supported  | API not built         |
        | **502**     | Bad Gateway           | Proxy got bad response | Nginx → Flask failure |
        | **503**     | Service Unavailable   | Service unavailable    | Server overloaded     |
        | **504**     | Gateway Timeout       | Upstream took too long | Database/API timeout  |

    General classification:
        | Range   | Meaning       | Simple Explanation                  |
        | ------- | ------------- | ----------------------------------- |
        | **1xx** | Informational | Request received, processing        |
        | **2xx** | Success       | Request completed successfully      |
        | **3xx** | Redirection   | Client needs to take another action |
        | **4xx** | Client Error  | Problem is with the request/client  |
        | **5xx** | Server Error  | Problem is with server/application  |
